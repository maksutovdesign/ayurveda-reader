# Кабинет экспертов — архитектура

Цель: эксперты входят через Telegram, предлагают правки/переводы глав и стихов,
администратор одобряет. Одобренные правки накладываются поверх статических данных книг.

## Принципы

1. **Аддитивность.** Статический сайт работает как раньше, даже если бэкенд/БД недоступны.
   Правки — это слой «overrides» поверх базовых данных в JS-файлах.
2. **Без паролей.** Авторизация через Telegram Login Widget (переиспользует существующего бота).
   Подпись проверяется на сервере по BOT_TOKEN (HMAC-SHA256).
3. **Stateless-сессии.** После входа сервер выдаёт подписанный токен (HMAC по BOT_TOKEN),
   хранится в localStorage. Никакого стороннего хранилища сессий.

## Роли

| Роль | Кто | Права |
|------|-----|-------|
| Гость | не вошёл | чтение |
| Пользователь | вошёл через Telegram | чтение + может предложить правку |
| Эксперт | tgId в наборе `experts` (KV) или `EXPERT_TG_IDS` | правки помечаются доверенными |
| Админ | tgId в `ADMIN_TG_IDS` (env) | одобрение/отклонение правок, управление экспертами |

## Хранилище — Vercel KV (Upstash Redis)

Ключи:
- `proposal:<id>` — объект правки `{id,tgId,tgName,bookId,sthana,chapter,verseNumber,field,oldValue,newValue,comment,status,createdAt}`
- `proposals:pending` — список id ожидающих
- `proposals:all` — список всех id
- `override:<bookId>:<sthana>:<chapter>:<verseNum>:<field>` — одобренное значение (живой слой)
- `experts` — набор tgId экспертов

## API (Vercel serverless)

- `POST /api/auth-telegram` — принимает payload Telegram Login, проверяет hash, возвращает сессионный токен
- `GET  /api/overrides?book=<id>` — одобренные правки книги (для наложения на статику)
- `GET  /api/proposals` — (админ) список ожидающих правок
- `POST /api/proposals` — (пользователь/эксперт) создать правку
- `POST /api/proposals/review` — (админ) одобрить/отклонить

Все эндпоинты деградируют мягко: если KV не настроен — возвращают пустой результат/ошибку,
фронтенд продолжает работать на статических данных.

## Настройка на Vercel (делает владелец)

1. Vercel → Storage → создать **KV** (Upstash Redis), привязать к проекту
   (автоматически добавит `KV_REST_API_URL`, `KV_REST_API_TOKEN`).
2. Env vars:
   - `ADMIN_TG_IDS` = ваш Telegram user id (узнать у @userinfobot), через запятую
   - `EXPERT_TG_IDS` = (опц.) стартовый список экспертов
   - `BOT_TOKEN` — уже есть (используется для проверки Telegram Login)
3. У бота в @BotFather: `/setdomain` → указать домен сайта (для Telegram Login Widget).
